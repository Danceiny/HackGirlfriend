## 协作项目目录结构说明

### Applications
单个应用以flask-blueprint来做，在Applications里做路由转发。

### Platform
Applications转发到此（有一个对应关系）。

### docs
项目中用到的技术，学习中可以留下**简要**的文档，并不断规范化。master分支下的该文件夹是GitHub Pages的目录，访问[HackGirlfriend Pages](https://cannot.cc/HackGirlfriend).

### Libraries
项目中用到的库（第三方或者自建）。

### PackageData
公用配置信息。诸如项目部署的配置文件。

### Spider
项目中用到的爬虫。

### Temp
临时代码。
